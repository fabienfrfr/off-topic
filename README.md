# Cheatsheet Pydantic

| Concept | Utilité | Exemple (1 ligne max) |
|---------|---------|------------------------|
| BaseModel | Créer des modèles de données | `class User(BaseModel): name: str; age: int` |
| Validation | Assurer l'intégrité des données | `age: PositiveInt = Field(..., lt=150)` |
| Types personnalisés | Contraintes spécifiques | `class ISBN(str): @validator('self') def check(cls, v): ...` |
| Config | Personnaliser le modèle | `class Config: allow_mutation = False` |
| Champs optionnels | Données facultatives | `email: Optional[str] = None` |
| Alias | Mapper les noms de champs | `name: str = Field(alias='userName')` |
| Alias camelCase | Conversion automatique | `class Config: alias_generator = to_camel` |
| Validateurs | Validation personnalisée | `@validator('password') def check_password(cls, v): ...` |
| Sérialisation | Convertir en formats standard | `user_dict = user.dict(); user_json = user.json()` |
| Désérialisation | Créer depuis données brutes | `user = User.parse_obj(data)` |
| Héritage | Étendre les modèles | `class Employee(User): department: str` |
| Modèles imbriqués | Structures complexes | `class Order(BaseModel): user: User; items: List[Item]` |
| Champs dynamiques | Structures flexibles | `class DynamicModel(BaseModel): __root__: Dict[str, Any]` |
| Génériques | Modèles typés réutilisables | `class Response(Generic[T]): data: T` |
| Schéma | Générer documentation | `schema = User.schema()` |

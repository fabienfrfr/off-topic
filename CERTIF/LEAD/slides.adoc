== Slide 2 : Architecture Microservices et IA
[.text-center]
*Intégration des modèles d'IA dans une architecture microservice*

---
**Schéma d'architecture :**
[kroki, type="plantuml", format="svg"]
----
@startuml
skinparam monochrome true

rectangle "API Gateway" as api
rectangle "Service A" as sa
rectangle "Service B" as sb
rectangle "Modèle IA" as ml

api --> sa : Requête utilisateur
sa --> ml : Appel au modèle
ml --> sb : Traitement des données
sb --> api : Réponse
@enduml
----

---
**Avantages :**
- Scalabilité
- Isolation des composants
- Facilité de maintenance

---

== Slide 3 : Défis et Bonnes Pratiques
[.text-center]
*Comment éviter les pièges ?*

---
**Défis courants :**
1. Latence des modèles
2. Gestion des données
3. Sécurité et conformité

---
**Bonnes pratiques :**
- Utiliser des **pipelines CI/CD** pour les modèles
- Monitorer les performances avec des outils comme **Prometheus**
- Documenter avec des outils comme **AsciiDoc** !

---
**Schéma de pipeline MLOps :**
[kroki, type="mermaid", format="svg"]
----

    A[Données Brutes] --> B[Prétraitement]
    B --> C[Entraînement]
    C --> D[Validation]
    D --> E[Déploiement]
    E --> F[Monitoring]
----
